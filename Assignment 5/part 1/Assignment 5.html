Assignment 5 – Web Accessibility

This document provides answers to the project brief questions for the accessibility troubleshooting challenge.

Color

The text was difficult to read because the white text on the green background had insufficient color contrast. The contrast ratio was approximately 2.57:1, which is below the WCAG AA standard of 3:1 for large text.

To fix this, I changed the background color from green (#008000) to #004000, resulting in a contrast ratio of about 4:1, which meets the AA standard.

Semantic HTML

Without proper semantic HTML, screen readers would struggle to navigate the content structure, as headings were simulated with font sizes rather than actual heading elements, and the navigation was in a generic div.

To improve navigation, I replaced <font> tags with proper <h1>, <h2>, <h3> elements, converted paragraphs from <br>-separated text to <p> elements, and changed the navigation div to a <nav> element.

The navigation menu was updated to use the <nav> semantic element for better accessibility.

CSS selectors were updated from font[size="7"] etc. to h1, h2, h3, and div[class="nav"] to nav.

The Images

The images lacked alt text, making them inaccessible to screen reader users.

I added descriptive alt attributes: alt="A wild bear standing in a forest" for the wild bear image and alt="An urban bear wearing a jacket in a city" for the urban bear image.

The Audio Player

The audio player was not accessible to hearing impaired users as it lacked any text alternative.

I added a <details> element with a transcript of the audio content.

For older browsers, the <audio> element includes a fallback message and a download link: <p>Your browser does not support HTML5 audio. <a href="media/bear.mp3">Download the audio file</a>.</p>

The Forms

The search input lacked a label, and the comment form labels were not properly associated with their inputs.

For the search form, I added a visually hidden label: <label for="search-input" class="visually-hidden">Search query</label> with CSS to hide it from sighted users.

For the comment form, I replaced the plain text labels with <label> elements using for and id attributes to associate them unambiguously.

CSS was updated to style the flex layout for the comment form pairs.

The Show/Hide Comment Control

The control was a div with onclick, which is not keyboard accessible.

I changed it to a <button> element, which is natively focusable and activatable with the Tab key and Return/Enter key.

The Table

The table lacked a summary and proper header associations, making it hard for screen readers to understand the data structure.

I added a <caption> element with "Bear facts table", changed header cells to <th> with scope="col" to associate columns.

Other Considerations

Two additional improvements:

  • Added a skip link at the top of the page: <a href="#main" class="visually-hidden">Skip to main content</a> to allow keyboard users to bypass navigation.
  • Ensured the HTML has lang="en" for proper language identification by assistive technologies.
